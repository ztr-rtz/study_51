## 1. CPU、MCU、MPU、GPU、NPU、FPU、SoC 各是什么？

**CPU**（Central Processing Unit）
纯处理器核心本身不带存储和外设，要靠外接 RAM、ROM、硬盘才能干活。比如你买一颗 Intel i7，光有它开不了机还得配主板内存条固态硬盘。速度快，能做复杂运算但自己什么都不带。

**MCU**（Micro Controller Unit 微控制器）
把 CPU + RAM + ROM + GPIO + Timer + UART + ADC 等全塞进一颗芯片里。相当于整台电脑缩到一块 IC 上。典型的就是 51 单片机、STM32 用来做简单控制，控制个 LED 读个按键温度传感器啥的。集成度高、引脚少、便宜、适合嵌入式控制。

**MPU**（Micro Processing Unit 微处理器）
本质就是 CPU 的升级版，处理器核心弄在一块芯片上但存储和外设要自己在外面搭。比如 ARM Cortex-A 系列的芯片跑 Linux，你得外接 DDR 内存、eMMC 闪存。性能强、能跑 OS、但得要外围电路配合。

**GPU**（Graphics Processing Unit）
图形处理器，内部结构是显存 + GPU 核心 + 散热 + 输出接口。专做图形渲染和图像处理，能同时并行算大量浮点运算。不管是打游戏还是跑 AI 训练，核心就是 GPU 在干活。NVIDIA 的 GeForce、RTX 系列就是典型。

**NPU**（Neural Processing Unit）
神经网络处理器专门为 AI 推理和卷积运算设计的。和 GPU 不一样的地方是 NPU 专为神经网络的矩阵乘法和激活函数做了硬件级优化功耗低效率高。手机里骁龙芯片内置的 Hexagon NPU、华为的达芬奇架构 NPU 就是干这个的。

**FPU**（Floating Point Unit）
浮点运算单元通常嵌在 CPU 内部，专门加速小数运算。没有 FPU 的话 CPU 只能整数运算，浮点要软件模拟特别慢。ARM Cortex-M4/M7 带 FPU，M0/M3 不带做浮点就费劲很多。

**SoC**（System On Chip 片上系统）
把多个不同功能的独立芯片集成到一颗芯片上的终极全家桶。手机上骁龙 8、苹果 A 系列芯片里面有 CPU+GPU+NPU+ISP+DSP+5G 基带+内存控制器。并不是把所有东西集成一起就叫 SoC，关键是集成的都是不同功能模块，每个拿出来都能独立干活。

一图总结：
```
SoC ─────────────────────────────────────────┐
│  CPU(大脑) │ GPU(画师) │ NPU(AI专才)        │
│  FPU(算小数) │ 内存控制器 │ 外设接口(USB等) │
└────────────────────────────────────────────┘

MCU = CPU + RAM + ROM + GPIO + Timer + 外设 → 一张芯片
MPU = 只有CPU核心 → 外接DDR + 外接Flash + 外接各种外设控制器
```

## 2. 单片机最小系统包括？

三个缺一不可的部分：

**电源电路**
VCC（正极）和 GND（地）给芯片供电。STC89C52 工作电压一般是 5V。供不上电一切免谈芯片连启动都启动不了。

**晶振电路**（时钟电路）
晶振 + 两个匹配电容（一般是 20~30pF），接在 XTAL1 和 XTAL2 脚上。晶振就是芯片的心跳给 CPU 提供时钟节拍——频率越高跑得越快。51 常见 12MHz 或 11.0592MHz（后者方便算波特率）。没晶振芯片就是个砖头时钟停摆一切指令都无法执行。

**复位电路**
上电时给 RST 引脚一个短时间的高电平把芯片内部所有寄存器拉回初始值，然后拉低让程序从地址 0x0000 开始跑。一般用 RC 电路（电阻+电容）实现上电复位，再并联一个按键做手动复位。没复位电路芯片状态不确定第一次上电可能飞到随机地址跑飞。

```
最小系统 = 电源(5V/GND) + 晶振(12MHz+两个电容) + 复位(RC+按键)
```

这三个搭好芯片就能跑代码了，LED、数码管这些外设是加到最小系统上的。

## 3. STC89C52 芯片内部结构？

STC89C52 是宏晶科技基于 8051 内核的单片机内部包含：

**CPU 核心**
8位 8051 处理器内核包含运算器 ALU、累加器 ACC、寄存器 B、程序状态字 PSW、堆栈指针 SP、数据指针 DPTR、程序计数器 PC。所有运算和逻辑控制都在这里完成。

**存储器**
- 8KB Flash ROM：放程序代码（ISP 在线编程烧写）
- 512 字节内部 RAM：256 字节常规 + 256 字节扩展 XRAM
- 4KB EEPROM：掉电不丢失，存配置参数

**I/O 端口**
4 组 8 位双向 I/O：P0、P1、P2、P3，每组 8 个脚共 32 个 GPIO。P3 口有复用功能如串口、外部中断、定时器引脚。

**定时器/计数器**
3 个 16 位定时器 T0、T1、T2。能做定时延时、计数外部脉冲、还用来产生串口波特率。

**串口通信**
1 个全双工 UART 串行口通过 P3.0(RXD) 和 P3.1(TXD) 通信。

**中断系统**
6 个中断源：2 个外部中断 INT0/INT1、3 个定时器中断 T0/T1/T2、1 个串口中断。两级优先级。

**时钟与复位**
内部振荡电路接外部晶振，加上复位逻辑。

**看门狗**
内置看门狗定时器程序跑飞了能自动复位防止死机。

```
┌──────────────── STC89C52 ────────────────┐
│  8051 CPU (8位ALU、ACC、B、PSW、SP...)   │
├──────────────────────────────────────────┤
│  8KB Flash ROM  │  512B RAM  │  4KB EEPROM │
├──────────────────────────────────────────┤
│  P0(8) │ P1(8) │ P2(8) │ P3(8)          │
├──────────────────────────────────────────┤
│  T0/T1/T2 3个16位定时器 │ 1个UART串口     │
├──────────────────────────────────────────┤
│  6中断源(INT0/1、T0/1/2、串口) │ 看门狗   │
├──────────────────────────────────────────┤
│  时钟振荡器(接外部晶振) │ 复位电路        │
└──────────────────────────────────────────┘
```

## 4. 内存和外存的特点？

**内存（RAM）**
- 速度快，CPU 直接按地址访问
- 掉电数据全丢
- 价格贵，容量越小越贵（按每字节算）
- 存的是运行时变量和临时数据
- 工作期间随时可读可写

**外存（ROM / Flash / 硬盘）**
- 速度慢，比 RAM 低几个数量级
- 掉电不丢数据，持久保存
- 便宜大碗，几十块就能买几百 GB
- 存的是程序代码和文件
- 运行前先把程序从外存加载到内存

一图看懂层次关系：

```
┌──── 寄存器(CPU内部) ── 最快，几KB，CPU直接访问
│  ┌── 缓存(Cache)  ── 很快，几十KB~MB，CPU和RAM之间
│  │  ┌── 内存(RAM)  ── 快，几百MB~几GB，放运行时数据
│  │  │  ┌── 外存(ROM/Flash/HDD)  ── 慢，几十GB~TB，放程序
▼  ▼  ▼  ▼
速度越来越慢，容量越来越大，价格越来越便宜
```

程序运行流程：
```
a.out(外存中) → 加载到RAM → CPU按地址从RAM取指令和数据 → 执行 → 结果写回RAM
```

## 5. RAM 和 ROM 的特性？

| 对比项 | RAM（随机存储器） | ROM（只读存储器） |
|--------|-------------------|-------------------|
| 全称 | Random Access Memory | Read Only Memory |
| 可写性 | 随时可读可写 | 正常运行时只读，写入需特殊方式（烧写/编程） |
| 掉电 | 数据全部丢失（易失性） | 数据保留（非易失性） |
| 速度 | 快 | 比 RAM 慢 |
| 用途 | 存放运行时变量、堆栈、临时计算结果 | 存放程序代码、固定数据表 |
| 在51中 | 512B 片内 RAM，存变量 | 8KB Flash ROM，烧程序进去 |

举例说明：
```c
int a = 2, b = 3;
int sum = a + b;
// a、b、sum 这三个变量的值存在 RAM 里
// 而 "int a = 2;" 这句对应的机器指令存在 ROM 里
// CPU 从 ROM 取指令 → 从 RAM 取 a、b 的值 → ALU 算加法 → 结果 sum 写回 RAM
```

SRAM vs DRAM 补充：
- SRAM（静态RAM）：用触发器存数据、快、贵、不用刷新，做 Cache 和片内 RAM
- DRAM（动态RAM）：用电容存数据、慢一点、便宜、需要定时刷新，做电脑内存条

51 单片机片内 RAM 是 SRAM。

## 6. CPU 与 RAM 之间三大总线及作用？

CPU 访问 RAM 靠三条总线配合完成。这三条线就是 CPU 和 RAM 之间的高速公路。

| 总线 | 方向 | 作用 | 在51上的宽度 |
|------|------|------|-------------|
| 地址总线 | CPU→RAM（单向） | CPU 发出要访问的地址告诉 RAM 要去哪找数据 | 16位，最大寻址 2^16=64KB |
| 数据总线 | CPU↔RAM（双向） | 按地址读写数据实际搬运数据的通道 | 8位，一次传1字节 |
| 控制总线 | CPU↔RAM（双向） | CPU 发读/写信号，RAM 回传就绪/忙信号 | 多条信号线 |

一次读取的工作流程：
```
1. CPU 把地址放到地址总线上 → 比如 0x0030
2. CPU 在控制总线上发出 读 信号 (RD=0)
3. RAM 根据地址找到对应存储单元
4. RAM 把该单元的数据放到数据总线上
5. CPU 从数据总线读走数据，读信号恢复 1
```

一次写入的工作流程：
```
1. CPU 把目标地址放到地址总线上 → 比如 0x0040
2. CPU 把要写的数据放到数据总线上
3. CPU 在控制总线上发出 写 信号 (WR=0)
4. RAM 把数据总线上的数据写入对应地址
5. 写信号恢复 1
```

## 7. 如何实现数码管的显示？

以开发板上 **4位共阴极数码管** 为例。

**硬件连接**
- P0 口：控制段选（a、b、c、d、e、f、g、dp 八个段）
- P2 口低 4 位（P10~P13）：控制位选（通过 NPN 三极管驱动）

**段选——决定显示什么数字**
8 段数码管，每段对应一个 LED。共阴极的意思是所有 LED 的负极连在一起接 GND。要点亮某一段只需把对应的引脚置高电平。

比如显示数字 "0" = 点亮 a、b、c、d、e、f，熄灭 g：
```
  a
f   b
  g
e   c
  d    dp

0 的字形码 = a+b+c+d+e+f = 0011 1111 = 0x3F
```

**位选——决定哪一位亮**
同时只有一位导通，通过三极管把对应位的共阴极拉到 GND。

显示一个数字的基本流程：
```
1. 关闭所有位选（消隐，防止残影）
2. P0 = 字形码（设段选）
3. 选通目标位（给对应三极管基极高电平）
4. delay(很短的时间)
5. 关闭该位选
6. 回到步骤1处理下一位
```

## 8. 数码管动态显示原理？

动态显示靠的是**视觉暂留效应**。

**为什么不能静态显示四位？**
如果四位数码管同时点亮——四位共阴极端子直接接到一起，段选信号相同，四个位会显示完全一样的数字。所以必须分时点亮。

**工作过程**
```
while(1) {
    关位选（消隐）
    把"千位"的字形码放到 P0
    开第1位位选 → delay(1ms) → 关第1位
    
    关位选
    把"百位"的字形码放到 P0
    开第2位位选 → delay(1ms) → 关第2位
    
    关位选
    把"十位"的字形码放到 P0
    开第3位位选 → delay(1ms) → 关第3位
    
    关位选
    把"个位"的字形码放到 P0
    开第4位位选 → delay(1ms) → 关第4位
}
```

**视觉暂留原理**
人眼对光的反应有约 50ms 的余辉。四位在一个循环里每位亮 1ms，完整循环约 4ms，频率约 250Hz 远高于人眼能察觉的闪烁频率（约 50Hz）。所以人看起来四位是同时在亮的。每个位虽然只亮了很短时间，但一轮刷新足够快眼睛就觉得一直是亮的。

**消隐的意义**
切换位选前必须先把位全关掉然后换段选数据，否则前一位的段码会短暂地出现在下一位上造成鬼影。消隐一步不可省否则显示会乱套。

## 9. 指定位清0和指定位置1如何实现？

51 单片机是 8 位机，最常用 `unsigned char`（1 字节 = 8 位）。假设：
```c
unsigned char t = 0x00;  // 0000 0000
```

### 指定位清0（置0）

核心：`t &= ~(1 << n)` —— 把第 n 位清 0，其他位不变。

原理：1<<n 造一个只有第 n 位是 1 的数，然后 ~ 取反把它变成只剩第 n 位是 0，再  &=  与运算。与 0 为 0 与 1 不变。

```c
t = 0xFF;               // 1111 1111
t &= ~(1 << 1);         // 把 bit1 清0 → 1111 1101 (0xFD)
t &= ~(1 << 3);         // 再把 bit3 清0 → 1111 0101 (0xF5)
```

### 指定位置1

核心：`t |= (1 << n)` —— 把第 n 位置 1，其他位不变。

原理：1<<n 只有一个位是 1，｜= 或运算，或 1 得 1、或 0 不变。

```c
t = 0x00;               // 0000 0000
t |= (1 << 0);          // bit0 置1 → 0000 0001 (0x01)
t |= (1 << 7);          // bit7 置1 → 1000 0001 (0x81)
t |= (1<<2) | (1<<3);   // bit2和bit3 同时置1 → 1000 1101 (0x8D)
```

速查表：
| 操作 | 写法 | 口诀 |
|------|------|------|
| 第n位置1 | `t \|= (1 << n)` | 左移 n 位然后或 |
| 第n位清0 | `t &= ~(1 << n)` | 左移 n 位取反然后与 |
| 翻转第n位 | `t ^= (1 << n)` | 左移 n 位然后异或 |
| 读第n位状态 | `(t >> n) & 1` | 右移 n 位看最低位是不是1 |

## 10. 什么是外设寄存器？

**一句话定义**：外设寄存器就是一段映射到固定地址的内存空间。你往这个地址写数据就等于在给外设硬件发指令。

芯片厂商把外设（GPIO、串口、定时器等）的控制信号通过地址线接到一段固定地址上。对 CPU 来说写一个外设寄存器和写一个内存地址的操作是一样的——都是"把数据放到地址总线上，把值放到数据总线上，发写信号"。

拿 P2 口举例：
```c
#define P2  *((unsigned char *)0xA0)

P2 = 0xFF;  // 往地址 0xA0 写 1111 1111 → P2.0~P2.7 全部输出高电平
P2 = 0x00;  // 往地址 0xA0 写 0000 0000 → P2.0~P2.7 全部输出低电平 → LED全亮
```

**内存映射 vs 独立编址**
51 单片机用的是**统一编址**（内存映射 I/O），即外设寄存器和普通 RAM 共用同一套地址空间。地址 0x00~0x7F 是普通 RAM，地址 0x80~0xFF 是特殊功能寄存器 SFR（外设寄存器区）。对 CPU 来说访问 RAM 和访问外设寄存器汇编指令完全一样都用 MOV。

各家芯片的外设寄存器不同，寄存器地址和各个位的含义要查该芯片的参考手册（datasheet）。

## 11. 控制 LED 和数码管分别用到了哪几个外设寄存器？

以普中 HC6800-MS 开发板为例：

**控制 LED**

LED 灯接了 P2 口 8 个引脚，低电平点亮（共阳极接法）。只用到一个寄存器：

- **P2 寄存器**，地址 0xA0
  - `P2 = 0xFF` → 8位全1全高电平 → 灯全灭
  - `P2 = 0x00` → 8位全0全低电平 → 灯全亮
  - `P2 = ~(1<<0)` → 只有 bit0 低电平 → 只有第1个LED亮

**控制数码管**

用到了两个寄存器：

- **P0 寄存器**，地址 0x80 —— 控制段选
  8 位分别对应 a、b、c、d、e、f、g、dp 段。共阴极高电平亮。
  显示 "0" 写 `P0 = 0x3F`（= 0011 1111 = a~f 全亮、g 灭、dp 灭）

- **P2 寄存器**，地址 0xA0 —— 控制位选（用低 4 位）
  P2.0→P1.0、P2.1→P1.1、P2.2→P1.2、P2.3→P1.3，接 NPN 三极管基极，高电平选通对应位。
  - `P2 = 0x01` → 选第1位数码管
  - `P2 = 0x02` → 选第2位
  - `P2 = 0x04` → 选第3位
  - `P2 = 0x08` → 选第4位

数码管动态显示 = P0（段选字形码）+ P2（位选切换）+ 定时刷新。

```
┌─────────────────────────────────────┐
│           LED 灯控制                 │
│  P2寄存器(0xA0) → 8个LED直接驱动    │
│  低电平点亮                          │
├─────────────────────────────────────┤
│         数码管控制                   │
│  P0寄存器(0x80) → 段选(a~g+dp)     │
│  P2寄存器(0xA0) → 位选(低4位选通)   │
│  共阴极高电平点亮                    │
└─────────────────────────────────────┘
```

注意：P2 口被 LED 和数码管位选复用了。所以开发板上 LED 和数码管通常不同时用——数码管动态扫描时会影响到 LED，LED 也会干扰数码管位选。
